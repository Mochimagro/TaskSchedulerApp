# Task Management App (MVP Architecture)
C#の専門性と、保守性の高いソフトウェア設計を実証するためのプロジェクトです。

## 💡 このプロジェクトの核心
単なる「タスク管理ツール」の制作ではなく、**「多人数開発や長期運用に耐えうる、疎結合なアーキテクチャの構築」**を主眼に置いています。

### 🔑 技術的ブレイクスルー：UIコンポーネントの完全疎結合化
当初、`TaskList`（一覧）と `TaskDetail`（詳細）の間に存在した直接的な参照（依存関係）を完全に排除しました。
- **解決策:** 相互参照を廃止し、Presenterを介したイベント通知モデルへ移行。
- **メリット:** 画面レイアウトの変更が他のコンポーネントに影響を与えないため、大規模開発における「並行作業のしやすさ」と「単体テストの容易性」を確保し、「役割を完全に離別して考えて、実装作業に当たることができる」ことを実現しています。

## 🏗 アーキテクチャ図

- **Model:** Pure C#。データの保持と永続化を担当。Unityから独立。
- **View:** UI表示に特化。唯一MonoBehaviorを継承できる。
- **Presenter:** 仲介者。UniRxを用いてViewのUI操作イベントやModelのデータ更新イベントを受け取り、各種処理をリアクティブに状態を伝播。

## 🛠 使用技術
- Unity 2022.3 (LTS) / C#
- **UniRx:** 非同期処理およびイベント駆動の実装
- **Zenject:** DI。外部代入機能による疎結合実現
- **MVP Pattern:** 責務分離の徹底
