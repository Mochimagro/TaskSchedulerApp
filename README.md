# Task Management App (MVP Architecture)

C#を用いた堅牢な設計と、uGUIによる直感的なUIを両立させたタスク管理アプリケーションです。
Unityを単なる「ゲームエンジン」としてではなく、「クロスプラットフォームなUIフレームワーク」として活用し、保守性と拡張性を重視した設計を実証しています。

## 🛠 開発の背景と目的
- **C#の専門性の追求:** MonoBehaviourへの依存を最小限に抑え、Pure C#に近い形でビジネスロジックを記述する設計を追求。
- **実用的な設計パターンの適用:** 複雑になりがちなUIの状態管理を、MVPパターンとUniRxを用いて整理。
- **モダンな開発フロー:** 疎結合な設計により、UI（View）の変更がロジック（Model）に影響を与えない開発構造を構築。

## 🏗 アーキテクチャ
本プロジェクトでは **MVP (Model-View-Presenter) パターン** を採用しています。

### 各層の役割
- **Model:** タスクのデータ構造、永続化（データの保存）、およびビジネスロジックを担当。Unity APIに依存しないPure C#で記述。
- **View:** uGUIを用いた表示とユーザー入力の受付。ロジックを持たず、Presenterからの通知を受けて表示を更新。
- **Presenter:** ModelとViewの仲介。UniRxを用いてModelの状態変更を検知し、Viewへ反映。また、ViewからのイベントをModelへ伝達。

### 採用テクノロジー
- **Unity 2022.3 (LTS)**
- **C# (C# 9.0+ 機能を活用)**
- **UniRx:** リアクティブプログラミングによるイベントハンドリング。
- **MVP Architecture:** 責務分離による保守性の向上。

## 🔍 技術的な見どころ
### 1. 徹底した疎結合設計
Presenterを介することで、View（画面レイアウト）とModel（データ処理）が完全に分離されています。これにより、将来的にWeb APIとの連携や、データベースの変更を行う際も、UI部分に影響を与えずに修正が可能です。

### 2. Reactiveによる状態管理
UniRxを活用し、データの変更を自動的にUIに反映させるリアクティブな設計を行っています。命令的な記述を減らすことで、バグの混入を防ぎ、コードの可読性を高めています。

### 3. スケーラビリティを意識したコード
将来的な機能追加（カテゴリー分け、通知機能、クラウド同期など）を想定し、Interfaceを用いた実装を行っています。
